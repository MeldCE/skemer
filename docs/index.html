options(schema, [baseSchema], [replace])

Options Object that must be passed to the one-off validate functions and on creating an instance of a Skemer

Parameters

  • schema schema :

    Schema to use for the validation

  • [Object] baseSchema :

    Schema to be used for recursive schemas. If none given, the given, the full schema given in schema will be used

  • [boolean or Array<boolean>] replace :

    A boolean to specify whether to globally replace all existing values for arrays and objects, or an object of string/boolean key/value pairs used to specify what variables(their name given as the key) should have their value replaced by default (a boolean value of true)

schema

Schema Object detailing the schema to be used for validating and merging data.

buildJsDocs(schema, options)

Build a JSDoc for a variable using the given schema.

Parameters

  • Object schema :

    An Object containing a valid schema

  • Object options :

    An Object containing build options

    • [string] options.name

      Name of the object documenting (will be prepended to any parameter names

    • [string] options.type (default 'prop')

      Specify what block tag should be used for the variables

    • [number] options.tabWidth (default 8)

      The width (number of characters) of a tab

    • [string] options.preLine

      String (normally indentation) to include before each line

    • [boolean] options.lineup (default true)

      Whether to line up text in a JSDoc block (eg @param) with the end of the block command

    • [number] options.wrap

      Number of characters to wrap the JSDoc lines at

Returns

string :

A string containing the JSDoc for the given schema
